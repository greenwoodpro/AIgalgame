测试流程

1. 模型列表准备

从 3 个平台选取 20 个模型，覆盖主流厂商（智谱、DeepSeek、Qwen、Moonshot、MiniMax、Meta、OpenAI OSS）。

2. 第一轮测试 (R1)

对每个模型发送测试 prompt，记录响应时间、返回内容、HTTP 状态码。超时 90 秒自动终止。

3. 间隔等待

每轮测试后等待 1 秒，避免触发限流。

4. 第二轮测试 (R2)

重复相同 prompt，检验稳定性和一致性。

5. 质量评分

对成功返回的内容按 5 维度评分标准打分，计算总分。

6. 数据汇总

汇总速度、质量、稳定性数据，生成报告。

已知局限性

  • 单 prompt 测试：仅使用一个 Galgame 场景 prompt，不能代表模型在所有场景下的表现
  • 2 轮样本量小：仅 2 轮测试无法反映长尾故障率，稳定性结论仅供参考
  • API 代理影响：通过 Cloudflare Pages 代理访问，网络延迟可能影响速度数据
  • 平台兼容性：魔搭社区 4 个模型因 API 格式问题失败，不代表模型本身能力差
  • max_tokens 限制：800 tokens 可能限制部分模型的表达完整性
  • 时间波动：测试在特定时间段进行，不同时段模型负载不同可能影响速度
建议改进：增加多 prompt 类型测试（闲聊/推理/代码/格式输出）；增加测试轮次至 5 轮以评估稳定性；加入异步并发测试缩短总耗时；对 API 格式不兼容的模型尝试适配不同的响应解析逻辑。
